# Release v0.4.4 — Local LLM backend (Ollama), git hygiene for host overlays, dev documentation

This release documents and tags work that **switches the experimental LLM interaction path** on (per upstream Cowrie **`docs/LLM.rst`**), **keeps host-specific proc overlay out of Git**, and adds **detailed devlogs** for operators and future you.

## Why

- **LLM backend:** Cowrie can run a **high-interaction** session where command output is generated by an **OpenAI-compatible** HTTP API instead of the emulated shell backend. For a **local** model, the documented approach is to point **`[llm]`** at a local server (example: **Ollama** on port **11434** with **`/v1/chat/completions`**).
- **Git hygiene:** **`honeyfs/proc/meminfo`** and **`var/memstate.json`** are **machine/runtime** artefacts; they should not churn in every commit. They are ignored; **`meminfo`** was removed from the index so clones are not forced to match the Pi’s RAM numbers.
- **Traceability:** **`README`**, **`CHANGELOG.rst`**, **`PERSONAL_CHANGELOG`**, **`RELEASE_NOTES`**, and **`DEVLOG`** stay aligned so GitHub tags map to a written record.

## Operator checklist (LLM)

1. **Ollama** (or any **OpenAI-compatible** server) must be **reachable** at the configured **`host`** + **`path`** before sessions will get model output. Verify with a **`curl`** **POST** to **`/v1/chat/completions`** (see **`DEVLOG_v0.4.4.md`**).
2. **`model`** in **`[llm]`** must **exactly match** a model you have (`ollama list`); the default in config is a **placeholder** for your install.
3. **Revert to emulated shell** (Pi ground truth, **`rpi_ground`**, etc.): set **`[honeypot] backend = shell`** and restart Cowrie. The LLM path does not use the same static shell/filesystem stack.

## Security

- **`[llm] api_key`** in config: for cloud APIs, treat as a secret. For Ollama on **localhost**, Cowrie still sends **`Authorization: Bearer …`**; use a throwaway value if the server does not require a real key.
- LLM **responses are logged** when **`debug = true`** in **`[llm]`**; keep logs on trusted storage.

## Compatibility

- **Requires** Cowrie’s LLM code path ( **`src/cowrie/llm/`** ) as shipped; no custom patches for Ollama beyond configuration.
